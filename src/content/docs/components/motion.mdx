`@lattice-ui/motion` is the shared place for animation behavior in Lattice UI.
When a primitive needs transition timing, reduced-motion handling, enter/exit
state, or interpolation, it should use the motion package instead of owning a
private tween model.

## How to apply it

Use [System](/components/system/) for app-wide theme and density, [Layer](/components/layer/)
for overlay mounting and stacking, [Focus](/components/focus/) for trapped scopes
and restoration, and Motion for animated state. Keeping those ownership lines
clear prevents `Dialog`, `Popover`, `Select`, `Tooltip`, and `Toast` from each
inventing different rules for the same lifecycle.

- Use presence motion when content appears and disappears, especially portal
  content and indicators with exit transitions.
- Use response motion when a mounted control settles between states, such as a
  [Slider](/components/slider/) thumb, [Switch](/components/switch/) thumb, or
  selected item background.
- Use feedback motion for brief press or focus accents.
- Put reduced-motion policy behind `MotionProvider` so components can respect
  one shared policy.

For layered components, animate a visual wrapper rather than the placement
owner. [Popover](/components/popover/), [Select](/components/select/), and
[Tooltip](/components/tooltip/) still need placement to come from
[Popper](/components/popper/); Motion should make the content feel responsive
without invalidating the anchor geometry.

## Consumer guidance

For [Dialog](/components/dialog/), keep overlay fade and panel reveal as separate
motion concerns. The dialog owns open state, dismissal, focus trapping, and focus
restoration; Motion owns the visual transition while those semantics stay active.

For [Popover](/components/popover/), [Select](/components/select/), and
[Tooltip](/components/tooltip/), let the content remain mounted while exit motion
runs. Do not unmount portal content immediately on close if the user expects an
exit transition.

For [Toast](/components/toast/), keep queue timing and visibility in the toast
provider, then use Motion for the surface response. For [Slider](/components/slider/)
and [Switch](/components/switch/), let the component package own state and let
Motion settle the moving part instead of deriving animation from raw pointer or
toggle events.
