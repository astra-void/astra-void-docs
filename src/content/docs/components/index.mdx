## How To Use These Pages

- Read `What It Is For` first to decide whether the package matches your problem.
- Use `State Model` to understand controlled vs uncontrolled ownership and focus or overlay behavior.
- Use `Key API` for the small set of props and parts that change integration decisions.
- Use `Composition Patterns` to choose a structure that matches a real screen.
- Read `Cautions / Limits` before committing to the package in a larger flow.
