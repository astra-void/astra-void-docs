<div class="docs-landing-intro">
  <p>
    Lattice UI is a headless-first component stack for Roblox product UI. Interaction contracts,
    focus behavior, layering, and state ownership stay in the package. Visual structure, styling,
    and screen composition stay in the app that ships the interface.
  </p>
  <p>
    The documentation is written to help with implementation decisions, not to sell a design
    system in the abstract. Start with one real screen, then move into the package pages that match
    the part of the flow you are refining.
  </p>

  <div class="docs-note">
    <p>
      Recommended first pass: installation, one practical tutorial, then the Style, System, and
      Layer foundations before deeper component pages.
    </p>
  </div>
</div>

## Reading Order

1. Set up the workspace and confirm the CLI flow.
2. Build the settings dialog to see one coherent screen instead of disconnected snippets.
3. Read `Style`, `System`, and `Layer` before moving into specific packages like `Dialog`, `Select`, or `Text Field`.
