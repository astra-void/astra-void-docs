    <p class="proof-kicker">Headless primitives</p>
    <p>
      State, focus, and interaction contracts stay in the package. Roblox
      instance structure and styling stay in your app.
    </p>
  </section>
  <section class="proof-item">
    <p class="proof-kicker">Style + System</p>
    <p>
      Tokens, recipes, density, layout primitives, and surfaces were designed to
      work as one stack instead of unrelated utilities.
    </p>
  </section>
  <section class="proof-item">
    <p class="proof-kicker">Overlay discipline</p>
    <p>
      Dialog, menu, popover, tooltip, select, and combobox build on the same
      layer model so stacking and dismissal remain consistent.
    </p>
  </section>

## 1-Minute Real UI

The fastest way to understand the library is to follow one screen from shell to finished interaction.

<div class="timeline-grid">
  <article class="timeline-step">
    <div class="timeline-index">01</div>
    <div class="timeline-body">
      <h3>Start from a system-driven shell</h3>
      <p>Use `SystemProvider` and a stable playground shell so theme, density, spacing, and screen structure already exist before components arrive.</p>
    </div>
    <img src="/images/lattice-ui/settings-dialog-frame-1.svg" alt="Playground shell with dialog scene selected." />
  </article>

<article class="timeline-step">
  <div class="timeline-index">02</div>
  <div class="timeline-body">
    <h3>Compose a real field surface</h3>
    <p>
      Add TextField, helper text, validation state, and action rows without
      locking yourself into a pre-skinned control.
    </p>
  </div>
  <img
    src="/images/lattice-ui/settings-dialog-frame-2.svg"
    alt="Dialog content with text field and action buttons."
  />
</article>

<article class="timeline-step">
  <div class="timeline-index">03</div>
  <div class="timeline-body">
    <h3>Open portal-based selection content</h3>
    <p>
      Select uses registered items, anchored overlay content, and outside
      dismissal while still letting the app own every visible host.
    </p>
  </div>
  <img
    src="/images/lattice-ui/settings-dialog-frame-3.svg"
    alt="Select options opened inside the dialog."
  />
</article>

  <article class="timeline-step">
    <div class="timeline-index">04</div>
    <div class="timeline-body">
      <h3>Finish with one coherent product screen</h3>
      <p>The end result is one settings dialog with shared visual language and stable interaction rules, not a pile of disconnected snippets.</p>
    </div>
    <img src="/images/lattice-ui/settings-dialog-frame-4.svg" alt="Final settings dialog and implementation notes." />
  </article>
</div>
